# 説明書 — py2to3-agent

この文書は、IT や AI に詳しくない方に向けて、このリポジトリ（プログラム置き場）が
何をするものかを説明するものです。専門的な言葉はできるだけ使いません。

## 一言でいうと

古い書き方の Python プログラムを、今の書き方に直す「翻訳係」と、
その翻訳が正しくできたかを自動で採点する「採点係」のセットです。

## 何の役に立つのか

Python というプログラム言語には、古い世代（Python 2）と今の世代（Python 3）があります。
古い世代は公式のサポートが終わっており、今のパソコンではそのまま動かせないことが増えました。
しかし、会社や個人の手元には、古い世代で書かれたプログラムがまだ多く残っています。

そこで、古いプログラムを今の書き方に直す作業（移植）が必要になります。
たとえるなら、古い方言で書かれた文章を今の言葉に訳し、意味が変わっていないかを
原文と照合する作業に似ています。

このリポジトリは、その「直す作業」を AI に任せたとき、
結果が本当に正しいかを人の目に頼らず機械的に確かめるための仕組みを提供します。

## 正しさをどう確かめているか

「直したプログラムが正しい」とは、ここでは「直す前と同じ結果を出すこと」を指します。
確認は次の手順で行います。

1. 練習問題（古い書き方のプログラム）ごとに、「正しい結果」をあらかじめファイルに保存しておきます。
2. 直した後のプログラムを実際に動かし、出てきた結果を保存済みの「正しい結果」と照合します。
3. 文字どおり同じなら合格、少しでも違えば不合格です（前後の余分な空白だけは無視します）。

この照合を行うのが、採点プログラム（eval フォルダの oracle.py）です。
合否は決まった手順だけで機械的に決まり、AI の自己申告や人の印象には頼りません。

さらに、採点係そのものが壊れていないかを確かめる自己点検の機能もあります。
正しい見本を採点させて合格になること、わざと壊した見本を採点させて
不合格になることの両方を、全問題について確認します。
「何を出しても合格にしてしまう採点係」になっていないかを点検するためです。

また、直したプログラムが止まらなくなった場合に備えて、
一定時間（10 秒）で打ち切って不合格にする安全装置も入っています。

## フォルダとファイルの案内

- README.md … 利用者向けの正式な説明書（技術者向け）。
- 説明書.md … この文書。専門用語を使わない説明。
- .claude/agents/py2to3-agent.md … 翻訳係（AI エージェント）への指示書。
- eval/oracle.py … 採点プログラム本体。
- eval/corpus/ … 練習問題集。問題ごとに「お題」「正しい結果」「正しい直し方の見本」が入っています。
- design/design.md … なぜこの作り方を選んだかの記録。
- .github/workflows/ci.yml … 変更のたびに自己点検を自動で走らせる設定。

## 用語の小辞典

- 移植 … プログラムを別の環境・別の世代の言語で動くように直すこと。
- エージェント … 指示書に従って作業を進める AI の仕組み。ここでは翻訳係。
- オラクル … 結果の合否を機械的に判定する採点係。
- golden（ゴールデン） … あらかじめ保存しておいた「正しい結果」のファイル。
- corpus（コーパス） … 採点に使う練習問題集。
- CI … 変更を保存するたびに、点検を自動で実行してくれる仕組み。
